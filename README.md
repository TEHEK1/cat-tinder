# Кототиндер (Cat Tinder) v2.0

Мобильное приложение для просмотра и оценки фотографий котиков с информацией о породах. Свайпайте котиков, изучайте породы и собирайте лайки!

## Реализованные функции

### ДЗ-1: Основной функционал
- **Главный экран со свайпами** — отображение случайных изображений котиков с названием породы
- **Свайп-жесты** — смахивание влево (дизлайк) или вправо (лайк)
- **Кнопки лайк/дизлайк** — альтернативный способ оценки котиков
- **Счетчик лайков** — отображение и сохранение количества лайкнутых котиков
- **Детальная информация о котике** — полноэкранный просмотр с описанием породы
- **Список пород** — просмотр всех доступных пород с краткой информацией
- **Детальная информация о породе** — описание и характеристики породы
- **Таб-навигация** — переключение между экраном свайпов и списком пород

### ДЗ-2: Кототиндер Про
- **Онбординг** — горизонтально пролистываемый сценарий из 3 шагов при первом запуске с анимированным котиком (bounce, scale, rotate при перелистывании)
- **Регистрация и вход** — локальная авторизация с хранением пароля в Keychain (flutter_secure_storage), валидация полей, сохранение сессии между перезапусками
- **Clean Architecture** — рефакторинг под слоистую структуру (Domain / Data / Presentation) с централизованным DI (get_it)
- **Unit-тесты** — 20 тестов на валидацию и логику авторизации (AuthValidator, AuthRepositoryImpl)
- **Widget-тесты** — 8 тестов на экраны логина и регистрации (валидация, успех, ошибка)
- **CI/CD** — GitHub Actions workflow: flutter analyze + unit-тесты + widget-тесты
- **Аналитика** — логирование событий регистрации, входа, онбординга (абстракция AnalyticsService)

## Архитектура

```
lib/
├── main.dart                           # Точка входа
├── core/
│   ├── di/service_locator.dart         # Централизованный DI (get_it)
│   └── analytics/analytics_service.dart # Сервис аналитики
├── domain/
│   ├── entities/                       # Модели (Cat, Breed)
│   ├── repositories/                   # Абстракции репозиториев
│   └── validators/                     # Валидация (AuthValidator)
├── data/
│   ├── constants/                      # API-константы
│   ├── datasources/                    # Источники данных (API, Keychain, Preferences)
│   └── repositories/                   # Реализации репозиториев
└── presentation/
    ├── app.dart                        # Виджет приложения и роутинг
    ├── screens/                        # Все экраны
    ├── widgets/                        # Переиспользуемые виджеты
    └── utils/                          # Утилиты (ErrorHandler)
```

**Направление зависимостей:** Presentation → Domain ← Data

## Тесты

Запуск всех тестов:
```bash
flutter test
```

Запуск unit-тестов:
```bash
flutter test test/domain/ test/data/
```

Запуск widget-тестов:
```bash
flutter test test/presentation/
```

## Сборка

API-ключ передаётся через `--dart-define`:
```bash
flutter build apk --release --dart-define=CAT_API_KEY=ваш_ключ
```
